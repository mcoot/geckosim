# Design decisions

Short ADR-style records of the key decisions made while designing the gecko sim. Each file captures one decision: context, the call, why, and what follows.

New decisions append a new numbered file. Updates to existing decisions either edit in place (refinement) or add a superseding doc (reversal) — superseded docs stay in the repo, marked accordingly.
